# Styles Structure

This directory contains the modular CSS files for the Pinbox Syncer plugin.

## File Organization

- **index.css** - Main entry point that imports all other style modules
- **layout.css** - Layout-related styles (page structure, containers)
- **components.css** - Reusable component styles (buttons, warnings)
- **modals.css** - Modal dialog styles (delete confirmation, etc.)

## Development

When developing, edit the individual CSS files in this directory:

```bash
src/styles/
├── index.css       # Import entry point
├── layout.css      # Layout styles
├── components.css  # Component styles
└── modals.css      # Modal styles
```

## Building

The build script automatically concatenates all CSS modules into a single `styles.css` file:

```bash
# Build styles only
npm run build:styles

# Full build (includes styles)
npm run build
```

## Adding New Styles

1. Add your styles to the appropriate module file:
   - Layout changes → `layout.css`
   - New components → `components.css`
   - Modal styles → `modals.css`

2. If you need a new category, create a new `.css` file and import it in `index.css`:
   ```css
   @import './your-new-file.css';
   ```

3. Run the build to generate the final `styles.css`

## Production

The generated `styles.css` in the root directory is what Obsidian loads. This file is automatically generated during the build process and should not be edited directly.
